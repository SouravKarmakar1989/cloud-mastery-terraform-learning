Hi, everyone. In this video we are going to talk about connection draining, which is a really useful feature for the application gateway. So connection draining simply is going to remove the backend pool target from receiving a future requests. And this happens when the backend the target is going through some planned service updates. Definitely the application gateway is not going to terminate the existing connections with a particular back end target. However, what's going to happen is that the application gateway is going to maintain the existing connections. However, it's not going to open any new connection requests with the back end targets. And there is one exception in here. If we have the cookie based affinity is enabled, then the application gateway will continue to forward the traffic to the de-registered instances. Also, I want you to be aware that the connection draining also applies to the backend instances that are explicitly removed from the backend pool. So I believe the cookie affinity and connection draining are really useful features for the application gateway and you should use them for your use case. That's all I have for you in this video.
