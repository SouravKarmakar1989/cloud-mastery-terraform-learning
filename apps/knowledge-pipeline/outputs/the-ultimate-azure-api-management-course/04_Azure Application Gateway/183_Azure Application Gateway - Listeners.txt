So simply the listener is checking for incoming connection requests for the application gateways on a specific port IP address and protocol. The IP address could be public or private IP address for the application gateway, and the protocol could be Http or Https. Also, Http two is supported on the application gateway. However, it is disabled by default, and when it comes to the listener types, there are two main types the basic type and multi-site listener. For the basic listener, this is when you want all of the traffic are going to be directed to the public or private IP address of your application gateway. And then according to the routing rules, you are going to allow them to the right backend pool. And for the multi-site listener, this is when we create a multi site like Contoso.com and Fabric.com. Both are hosted on the application gateway and using the routing rules, we have been able to route the traffic to the correct backend. Pool. Order of processing is very important for version two. Multi site listeners are always processed before the basic listeners. However, this is not the case for version one where the listeners are going to be processed in order. And for the supported protocols you can use for your listeners, you can use Http or Https and we are going to see different scenarios for SSL termination and end to end SSL encryption for your application gateway.
