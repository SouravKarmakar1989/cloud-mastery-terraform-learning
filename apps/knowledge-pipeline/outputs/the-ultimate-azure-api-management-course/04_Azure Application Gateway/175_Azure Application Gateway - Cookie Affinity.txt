Hi, everyone. In this video we are going to talk about the cookie affinity for the application gateway and you can configure it in the Http settings for your application Gateway. Simply, the cookie affinity is a way to maintain user sessions. So when the user sends their first request to the application gateway, what's going to happen is that the application gateway is going to forward or route that request to a specific backend server. And then what we want to achieve here is when the user sends a subsequent requests to the application gateways, we want the application gateway to send those requests to the same server where the user had the session opened before. This is what we call it session stickiness sometimes. And we are able to achieve this by having a hash value in the response with the session details. So when the user sends future requests to the application gateway, the application gateway will be able to determine what was the back end server that the user session is stored on it. However, if your backend servers cannot handle cookie affinity, then you cannot use this feature obviously. Also, for more complex scenarios or more high availability scenarios, some people would prefer to have the user sessions stored on an external cache like Reddis or maybe on Cosmos DB to have the user sessions maintained outside of the backend servers completely. This is just to give you an idea of what is the cookie affinity is about and how you can set it up. Let's look at the Http settings in here. You can enable cookie affinity from the Http settings in the application gateway. That's all I have for you for this video. Thanks for watching and I will see you in the next video.
