Hi everyone. We have seen already in previous videos how we can integrate our Azure function with other Azure services like Service Bus, Cosmos, DB, SQL database and storage accounts. And the best way for you to integrate and connect your function app with other Azure services is by using triggers and bindings for your function app. Having said that, Azure function, executions by default are stateless. And if you have a requirement to implement it in a stateful way, then you need to consider using either durable functions or logic apps. Coming back to the trigram bindings when we say that the trigger and binding are the best way and easiest method for you to integrate your function app to other Azure services. This is not the only way. However, if you wish to have more control on the service connection and you are more comfortable using a client library provided in the service SDK, then you can do that on your own. However, if you are going to be using the client library, you need to pay more attention to the effect of scale and performance on the client connection. That's all I have for you in this video. It was a quick one to let you know about the stateless execution of the Azure functions and the different ways to integrate with the other Azure services other than triggers and bindings, of course.
