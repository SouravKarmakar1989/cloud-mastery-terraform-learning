Now that we understand the difference between managed and external tables, let's have a think about whether to use managed or external tables for our project. As we know, the raw data is manually imported in our solution. Even if we had an automated solution, that's unlikely to be Spark or Databricks. The most likely solution would be to use Azure Data Factory copy activity to pull the data from the Ergast API. Because Spark is not managing the data for this layer. The most appropriate solution is to use external tables. On the other hand, the data in Ingested and Presentation layer are managed by Spark for our project. We're not using any other solutions to create or update data in these layers. So the most appropriate solution is to use managed tables. Having said that, I've seen many industry projects using external tables for all three layers, so please don't be surprised to see that. The solution I propose here is based on the criteria that we discussed in the previous lessons for using external and managed tables. In the next few lessons, we'll create the external tables for our raw layer and change the PySpark notebooks which we already have for our ingestion and the presentation layers to create the managed tables while the data is being loaded. So let's get started.
