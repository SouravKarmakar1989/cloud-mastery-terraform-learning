So now let's talk about Time To Live, or TTL, in DynamoDB. So it's a feature that allows you to automatically delete items after an expired timestamp. And this timestamp must be an attribute from within the item, and it should be a number. And that number represents a timestamp in Unix Epoch format. And so what happens is that whenever the current time goes over that number, then the item will be marked for expiration and then deleted within a few days after expiration. So here we have an example of a session data table where we have User_ID and Session_ID as attributes. And then we've created an extra attribute called ExpTime for expiration time. And so we enter some data there, and this data is representing a timestamp in the Unix Epoch format. So over time, the current time will be evaluated and the Epoch timestamp of course, is going to increase of the current time. And so at some point the table will be scanned and for each items where the expired time column is less than the current Epoch time, then the items will be marked for expiration. And then there will be a second process, a deletion process that's going to scan the table again and delete the items that have been marked for expiration. So this Time To Live feature is really great and has a few use cases. The first one would be to reduce stored data by only keeping current items or to adhere to regulatory obligations or to have security features and guarantees that, for example, no data can be older than 30 days in your table.
