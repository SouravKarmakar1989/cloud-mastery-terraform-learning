# Demo - Static Website Hosting

## Pointwise No-Loss Summary

1. Demo builds baseline S3-hosted static site from bucket creation to public endpoint validation.
2. It demonstrates required permission changes for website object access.

## Steps Demonstrated

1. Create website bucket.
2. Upload full static site asset set.
3. Update bucket public-access settings to permit intended public website behavior.
4. Add bucket policy allowing public read of website objects.
5. Enable static website hosting in bucket properties.
6. Set index document configuration.
7. Open generated website endpoint and validate site renders.

## Outcome

1. Learner can launch functional static website directly from S3 bucket endpoint.
