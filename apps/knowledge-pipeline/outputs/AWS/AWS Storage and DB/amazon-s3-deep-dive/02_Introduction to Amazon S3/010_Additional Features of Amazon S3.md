# Additional Features of Amazon S3

## Pointwise No-Loss Summary

1. Lecture previews major S3 features covered later in course.
2. Focus areas include storage class choices, automation, security, access methods, and integrations.

## Storage Classing and Lifecycle

1. S3 offers multiple storage classes for different access/retention patterns.
2. Lifecycle policies can automatically:
   - transition objects between classes,
   - expire/delete objects by rule.

## Versioning

1. Object versioning stores multiple revisions of same key.
2. Helps recover from accidental overwrite/deletion and supports rollback patterns.

## Access Methods Mentioned

1. AWS Console.
2. Public AWS APIs.
3. AWS CLI.
4. SDK-based application access.
5. Access through other AWS compute/services (EC2/Lambda/etc.).
6. Private access patterns via VPC endpoint.
7. Access points and object URLs.

## Security Features Previewed

1. IAM policies/permissions.
2. ACL controls.
3. Encryption at rest options.
4. Object Lock and legal hold patterns.
5. Block Public Access controls.
6. Auditing and event-based controls.

## Other Feature Areas Mentioned

1. Static website hosting via S3.
2. Requester pays model for request-cost shifting.
3. Monitoring/auditing/event integrations.

## Key Takeaway

1. S3 is not only storage; it is a rich platform with lifecycle, security, access, governance, and hosting capabilities.
