In this video, we'll learn how we can use parameter groups to configure settings on our database engines. So let's start out by just understanding RDS is a managed service. You do not have direct access to the operating system of your RDS instance. You can't get shell access. You can't make configuration changes that way. But you may want to perform certain database configuration tasks that would normally require access to the operating system. So for example, MySQL has a my.cnf file that you can use to modify the configuration of the MySQL database engine. And using that configuration file you could do things like set the time zone, you could enforce encryption in transit. You can set. Different configuration options on the database engine itself. However, within RDS you don't have access to that configuration file. So instead we will use a parameter group to configure those sorts of settings. Every instance comes with a default parameter group. You can create a new parameter group and customize it with whatever settings you want, and you may have certain settings that you want to apply to multiple instances. Well, that's fine. When you create a parameter group, it can be applied to many instances. So for example, one of the settings that we might want to configure with a parameter group is to enforce SSL connections. This ensures that data that's transmitted between our client and our database is more secure, because all that traffic will be encrypted. It also ensures that the client that's connecting is an authentication server, and that it's not a man in the middle attack. So this is one of those security settings that we probably want to configure. And using a parameter group is a great way to do that. And in a few lessons we'll take a look at a demo of how to set up these parameter group settings.
