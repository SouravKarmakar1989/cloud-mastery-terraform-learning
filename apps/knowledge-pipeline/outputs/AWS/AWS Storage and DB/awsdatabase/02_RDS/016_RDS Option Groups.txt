In this video, we'll learn about option groups, and option groups is essentially a bundle of features or settings that you can apply to your RDS instance. These are options that are not included in the parameter group. Option groups are used for enabling and disabling specific database engine options like. Here's a good example. Maybe I want to create a memory cache and use a cache to improve the performance of my database. That's something that I would configure in an option group. Option groups are local to a specific region and local to the VPC that I create them in. Now this is important. If I ever need to restore the RDS instance to a different region. The option group that I used in the source region is not going to be available there. So the option group is like selecting a package of high level features for the RDS instance. Parameter groups are more like making detailed database specific configuration adjustments, but we really need both of them to make sure that we're tailoring the RDS database to meet the application's needs.
