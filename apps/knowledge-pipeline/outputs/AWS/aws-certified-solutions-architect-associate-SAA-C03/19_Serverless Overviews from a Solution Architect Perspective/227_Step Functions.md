# Step Functions

## Pointwise No-Loss Summary

1. AWS Step Functions builds serverless visual workflows for orchestration.
2. Commonly orchestrates Lambda but supports many AWS and external integration points.

## Workflow Capabilities Mentioned

1. Sequencing and branching.
2. Parallel execution.
3. Conditional logic.
4. Timeouts.
5. Error handling.
6. Human approval checkpoints.

## Integration Breadth Mentioned

1. Lambda.
2. EC2.
3. ECS.
4. On-premises systems.
5. API Gateway.
6. SQS.
7. DynamoDB (and broader AWS service ecosystem context).

## Example Use Cases

1. Order fulfillment.
2. Data processing.
3. Web application workflows.
4. Any complex multi-step process better represented as directed graph/state machine.

## Outcome

1. Step Functions is framed as orchestration layer for reliable stateful serverless workflows with built-in control-flow and failure-management semantics.
