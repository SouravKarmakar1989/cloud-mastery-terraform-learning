Say we have an EC2 instance with Port:22 open. We define a Config rule and then the resource itself, the security group will be marked as non-compliant. From then, we know we can invoke an SSM automation. So we're going to trigger it from Config and we're going to use the one provided by AWS out of the box called the AWS-DisableIncomingSSHOnPort22, which is going to disable the port on the security group. And this is done all out of the box from Config. Here's another example. We can enable logging on an S3 bucket. So again, Config will have a rule to monitor the non-compliant S3 buckets. And if one is non-compliant, we'll trigger an SSM automation again out of the box named AWS-ConfigureS3BucketLogging, in which we're going to enable logging on the bucket and we'll be good to go.
