So let's talk about the Default Host Management Configuration feature for Systems Manager or DHMC. The idea is that we know that to enable access to Systems Manager, our EC2 instances must have an IAM role with SSM access. But it turns out that you can also do without. So if you configure DHMC, then your EC2 instances can be automatically managed instances without the use of an EC2 instance profile. So how does this work? Well, we have an instance identity role, which is a type of IAM role that has no permissions. And this is for every EC2 instance that you create but you don't control it. And the role of this identity role is to just identify the EC2 instance to some AWS services, such as Systems Manager. So this instance identity role is different from your EC2 instance profile, okay? It's just something that is on every EC2 instance but you don't have control over it. But what's going to happen is that thanks to this instance identity role, the EC2 instance identifies with Systems Manager, which in return is going to pass an IAM role to the EC2 instance, which will allow the EC2 instance to be managed by Systems Manager. And again, all of this is done outside of the EC2 instance profile. So for this to work, you need to have the metadata version two enabled and the SSM agent enabled that is updated enough so it doesn't support IMDSv1. Once the instance is managed by Systems Manager, it will automatically enable all the features of Systems Manager, such as Session Manager, Patch Manager, and Inventory. And on top of it, the SSM agent will be automatically kept up to date. This must be enabled on a per region basis. Okay, so that's it for this lecture.
