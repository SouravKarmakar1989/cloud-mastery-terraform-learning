Now, let's talk about Amazon SNS and the Dead Letter Queues. So the idea is that you may have multiple subscribers to an SNS topic, for example, an HTTP endpoint or a Lambda function. And sometimes the delivery is not successful. That means that the messages are being retried, retried, retried, but then it doesn't work still. So the delivery retries have been exhausted. And what's going to happen is that the message is discarded, unless you set a Dead Letter Queue, or DLQ. So for this, you set a Redrive Policy, and you say, "Hey, if the message is discarded, instead please send it to a Dead Letter Queue." So it turns out that you define one Dead Letter Queue, one Redrive Policy per subscription on your SNS topic. So it's not one DLQ for your SNS topic, it's per subscription. So that means that your HTTP endpoint subscription will have its own DLQ, and then your Lambda subscription is also going to have its own DLQ. And that's it.
