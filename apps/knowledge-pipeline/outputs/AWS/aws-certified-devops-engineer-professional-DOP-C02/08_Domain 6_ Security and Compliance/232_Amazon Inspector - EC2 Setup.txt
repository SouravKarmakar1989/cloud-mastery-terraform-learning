So now let's have a detailed look at how we can enable Amazon Inspector for your EC2 instances. So it turns out that, first you must have the SSM Agent for your EC2 instance. That's because the Amazon Inspector Service actually leverages behind the scene the Systems Manager Service and for your EC2 instance to be managed by the Systems Manager Service, of course, step one is to have the SSM Agent but step two is also to have either an IAM role that has SSM Systems Manager permissions or enable default host management configuration so that your EC2 instance can become a managed instance of Systems Manager. Once this is done and that your instance is capable of communicating with the Systems Manager endpoint, either the public one or a private one using a private link, then Amazon Inspector can analyze the inventory results of the SSM Agent, and give you recommendations on your EC2 instance.
