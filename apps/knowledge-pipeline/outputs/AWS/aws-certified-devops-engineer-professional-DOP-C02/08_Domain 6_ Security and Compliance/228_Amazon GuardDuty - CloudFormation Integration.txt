So here is a short lecture on Amazon GuardDuty and CloudFormation. So of course you can enable GuardDuty from a CloudFormation template, but there is a problem. If GuardDuty is already enabled in the target account then your CloudFormation stack is going to fail. So how can we solve that problem? Well, we can use custom resources using a Lambda function, so that the Lambda function is going to enable GuardDuty only if it hasn't been enabled yet. So the idea is that we create a stack and the CloudFormation stack creates a custom resource, and this Lambda function will execute code, and will enable GuardDuty only if it is not enabled. So this solves this problem and this is an exam question. And so once we have this we can deploy this CloudFormation stack across all your organization using a stack sets.
