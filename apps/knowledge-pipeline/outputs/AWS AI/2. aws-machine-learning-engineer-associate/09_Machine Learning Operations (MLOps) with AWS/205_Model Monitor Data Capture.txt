Quick slide here on model monitor data capture, which is a feature of SageMaker model monitor. What data capture does is, well, what it sounds like. It logs all the input to your endpoints and inference outputs. So if you want to log everything that's coming into your inference endpoints and log what comes out, it can deliver that to S3 as JSON data and store that for you for future use, whatever you want to do with it. So model monitor data capture does exactly what it says. It captures the data going in and out of your inference endpoints and stores that off somewhere. What can you use that for. So you might want to use that for further training. Maybe that's a good way to learn over time what new data is coming into your system from the real world and augmenting the training of your model from that data. So model monitor can be part of the whole feedback loop of feeding new information back into your model for continuous training. It can also be useful for debugging and monitoring purposes obviously as well. It can also automatically compare your metrics associated with the data that's being captured to some sort of a baseline. So if you want to say, hey, maybe things are going a little bit out of bounds here, things are getting unusual. It can give you that comparison metric to see how things are changing over time. Data capture is supported for both real time and batch model monitor modes, so whether it's a real time predictions or batch model predictions, you can capture that data either way with Model Monitor, and it is supported for both Python using the Boto library, and also with the SageMaker Python SDK. You also have the option to encrypt that data of course. So we're always thinking about security with Amazon. So there is an option in data capture to encrypt that inference data as well.
