### Lecture: Retrieval-Augmented Generation (RAG) Fundamentals with Bedrock

Let's talk about a popular alternative to fine tuning retrieval, augmented generation, or Rag for short. And in bedrock we call that a knowledge base. So knowledge base is what we're covering here. But to understand knowledge bases you need to understand rag. So let's talk about retrieval augmented generation at a higher level. First to understand that you can think of Rag as sort of an open book exam for large language models. So the general idea is that we have some external database. We're querying for the answers instead of relying on the LLM. So you know, it's kind of a cheat to win approach here. Right. At a very high level. What happens is you have a prompt in this example to the right. What did the president say in his speech yesterday? Well, maybe your underlying foundation model doesn't know the answer to that because that was after its training data cut off. But you could say, okay, I'm going to go search my external database full of news articles for what did the president say in his speech yesterday, and try to find articles that are relevant semantically to that query. And then I'm going to get back maybe the text of that article that does talk about the president's speech yesterday and incorporates that into the prompt itself. So that would transform the prompt into something like, what did the president say in his speech yesterday? Consider the text of the following news article in your response. Last night president whatever said whatever, right? So you can see it's a pretty simple approach. At the end of the day, we're just saying, okay, we're going to take our prompt. I'm going to augment that prompt with data that I retrieved from some external database and include that in the prompt itself. So that's the high level idea. You just work those answers from the database or in the language of bedrock, a knowledge base into the prompt itself. And that gets passed on into the underlying foundation model. There are also ways to incorporate tools and functions to augment this knowledge base into the system. We'll talk about that when we talk about LM agents. And that's a slightly more principled way to go about this. So there are pros and cons to using retrieval augmented generation. Let's start with the pros because well that's what Amazon wants you to focus on because they make money. If you use generative AI and use these systems, it is faster and cheaper, at least at least in the short term, to incorporate new or proprietary information using Rag versus fine tuning. Uh, you don't have to fine tune your entire model every time you get new information. You can just expand your database. You're querying for this system instead, right? So that's a very easy thing to do. As new data comes in, you don't have to go back and do this expensive fine tuning operation. You can just incorporate that right into the database and you're done. Right. So it's a very good way to keep data up to date. However, it does increase the number of tokens in your prompts, right? Because you're including that external data into every prompt. And that can add up over time as well. So sometimes it's a little bit of a false economy. Updating information, like I said, is just a matter of updating a database. So very, very simple to keep your data up to date. For a generative AI system that includes Rag, it can also take advantage of what we call semantic search. So a very popular choice for the database in a retrieval augmented generation system is something called a vector database. We'll talk about that in a moment. ### Lecture: Other Amazon Bedrock Features (Model Evaluation, Bedrock Studio, Watermarks)

A few more bedrock features that we didn't talk about previously that are worth a mention. One is that, you know, again, you're not limited to the foundation models that bedrock provides or the custom models that you build on top of them. You can also import your own models that were trained from scratch. So in SageMaker you have that capability, right? You can go off and create your own foundation model and even train your own large language model from the ground up if you want to. You can import that model into bedrock. So you can either import that from SageMaker, or you can have a stored version of that model in S3 that you can import as well. So that's just a capability that bedrock offers. Also something worth talking about in some length is model evaluation. So if I do build a model or if I'm trying to compare models against each other, how do I measure them? How do I know one is better than the other? That's actually very hard to do with generative AI because the results are so non-deterministic sometimes, right? How do you decide what's good? Oftentimes it's a human decision, and that is one way to do it. But there are automatic metrics as well. So you can try to measure the accuracy toxicity Robustness. There's a metric called Bert score. You might want to remember F1, whatever it might be, and it can automatically compute those based on test data sets that you provide of prompts and the ideal responses to those prompts, basically. And you can either provide your own set of test prompts and responses, or there are built in prompt data sets you can use as well that have sort of a pre-built standard test set that will measure the relevancy of a given result from a model for a set of questions. And some of these data sets contain questions or prompts that are specifically designed to test the model's ability to reason things like that. So that's one way you can evaluate models within bedrock. Also, you can do it with humans. So if you want to bring your own team, it provides a framework for actually evaluating your models against human beings who are evaluating the quality of those responses. So for example, our self reinforcement learning from human feedback is an example of that, where you can put a couple of different responses from a model next to each other and ask humans to rate which one is better, right? You've probably seen that in other generative AI systems. That's what's going on there. It's basically human evaluation of the response from your model. And you don't have to bring your own team either. You can also use AWS managed teams as well, kind of like they do with Mechanical Turk. Right? So they have sort of this army of evaluators out there, human evaluators that you can leverage to evaluate your model and decide whether a given response from that model is good or not. Also, you have the ability to provide provisioned throughput. We talked about that briefly with LM agents. But more generally, if you want to make sure you have enough capacity for a given steady load, you have the ability to provide provisioned throughput for the inference layer in bedrock. Also. One other thing is watermark detection. So if you use the Titan Image Generation Foundation model from Amazon, it embeds a watermark into its results that can be detected. So if you want to take an image and determine if it was generated by that model, the flip side of that is that there's a watermark detection module in bedrock that can be used for that purpose. Also last thing is Bedrock Studio. This is kind of a new feature here, at least as of this recording. It's a web app environment. So it's a web based workspace for bedrock that allows people to use bedrock and develop within it without having an AWS account necessarily. So if you want to use bedrock within an organization and you don't want to have people to have their own AWS accounts to use it, Bedrock Studio is a way to do that from a security standpoint, uses some sort of single sign on integration with IAM and your own identity provider. So basically it ties together whatever sign on or account mechanism you have within your organization, and it integrates that with IAM under the hood. So it kind of looks like they're an AWS account under the hood. Using Bedrock Studio, different users can collaborate on projects and also components within that project, but they don't necessarily have to have their own AWS account. So good tool for collaboration within a larger environment.
